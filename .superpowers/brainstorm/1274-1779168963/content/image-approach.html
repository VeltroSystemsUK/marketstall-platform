Image sourcing for the new templates

How should images be populated in TRADES, WELLNESS, and HOSPITALITY demos?

A

Gemini generation (current approach)

Same as food stalls — Claude returns prompts, Gemini generates images, they're committed to the repo as static files.

Pros

  • Already wired up — zero new API keys or code
  • Images are truly custom, not stock

Cons

  • Gemini was tuned for food — quality varies for a plumber's van or a spa treatment room
  • AI-generated business photos look uncanny up close
B

Unsplash/Pexels API with AI keywords

Claude returns image_search_keywords per image slot. The deploy pipeline hits Unsplash (or Pexels) and commits the downloaded photo.

Pros

  • Real photography — looks polished on a tablet demo
  • Massive library for trades, wellness, hospitality
  • Free tier covers demo volume easily

Cons

  • One new API key (Unsplash or Pexels)
  • Stock photos feel less bespoke than generated ones
C

Hybrid — scrape client's own images first, Unsplash fallback

Try to use images scraped directly from the client's existing site. Where those are missing or low quality, fall back to Unsplash keyword search.

Pros

  • Demo uses the client's actual photos — most impressive to the client
  • Graceful fallback means the demo always looks complete

Cons

  • Legacy sites often have compressed, badly cropped images
  • Slightly more logic in the pipeline
