bizcustomizer — app structure & build order

Identical functionality to marketstall-customizer, adapted for service business sectors

File structure

# New standalone app
bizcustomizer/
  app/
    page.tsx          ← UI (same as stall)
    api/customize/
      route.ts      ← scrape+generate+images
      deploy/
        route.ts   ← commit+Vercel trigger
  lib/
    sectors.ts       ← TRADES/WELLNESS/HOSP config
    agent.ts        ← scrape, generate, images
    github.ts       ← copied from stall (unchanged)
    unsplash.ts     ← new: keyword → image
Same as marketstall-customizer:
  • Auth (ADMIN_TOKEN)
  • Streaming NDJSON API
  • Review → Deploy flow
  • Octokit GitHub commits
  • Firecrawl scraping
  • Gemini image generation
  • Claude relevance check
New in bizcustomizer:
  • Sector dropdown (not stall)
  • 3 sector-specific LLM prompts
  • 3 sector output schemas
  • 3 buildSiteConfig() builders
  • Unsplash fallback (unsplash.ts)

Build order

1
bizcustomizer scaffold — Next.js app, auth, UI shell, streaming API skeleton, github.ts, env vars
2
biztemplate-trades — Build the single-page Next.js template, define TradesOutput schema, write LLM prompt + builders, wire into customizer, end-to-end test
3
biztemplate-wellness — Multi-page template (/, /services, /team, /book), WellnessOutput schema + prompt + builders
4
biztemplate-hospitality — Multi-page template (/, /menu, /book), HospitalityOutput schema + prompt + builders
5
unsplash.ts fallback — Add Unsplash keyword search as final fallback in image pipeline (needs UNSPLASH_ACCESS_KEY)

Environment variables needed

ADMIN_TOKEN            ← same as stall customizer
ANTHROPIC_API_KEY      ← same
FIRECRAWL_API_KEY      ← same
GEMINI_API_KEY         ← same
GITHUB_TOKEN           ← same
UNSPLASH_ACCESS_KEY    ← new (free tier)
